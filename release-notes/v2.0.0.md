# Release v2.0.0

## Overview

This is the first release under the new versioning scheme (v2.x), marking a fresh start for the MCP Gateway Registry project.

## Highlights

### Core Platform
- **Unified MCP Gateway & Registry** - Centralized access point for MCP servers and AI agents
- **Agent-to-Agent (A2A) Communication Hub** - Agent registration, discovery, and direct communication
- **Semantic Search** - AI-powered tool discovery with vector embeddings

### Authentication & Security
- **Multi-provider OAuth 2.0/OIDC** - Support for Keycloak, Amazon Cognito, and Microsoft Entra ID
- **Security Scanning** - Integrated scanning for MCP servers and A2A agents
- **Fine-grained Access Control** - Multi-tenant permissions with scope-based authorization

### Infrastructure
- **Modern Python Stack** - Python 3.11+ with FastAPI, async/await patterns
- **Flexible Storage** - MongoDB Community Edition (local) / Amazon DocumentDB (production)
- **Container-first** - Docker and Podman support with pre-built images
- **Infrastructure as Code** - Terraform modules for AWS ECS deployment

### Developer Experience
- **AI Coding Assistant Integration** - Works with VS Code, Cursor, Claude Code, Cline
- **Comprehensive Testing** - pytest with parallel execution and coverage reporting
- **Modern Tooling** - ruff for linting/formatting, mypy for type checking, bandit for security

### Frontend
- **React 19** with TypeScript 5.9
- **Vite 6** for fast builds
- **Tailwind CSS 4** for styling

## Recent Improvements

- Enhanced `build_and_run.sh` with modular functions, strict error handling, and portable macOS/Linux support
- Global LLM configuration with fallback hierarchy for embeddings and security scanners
- Security audit and dependency updates (CVE patches)
- Pydantic v1 to v2 migration completed
- Comprehensive Helm charts for Kubernetes deployment

## Getting Started

```bash
# Clone and setup
git clone https://github.com/YOUR_ORG/mcp-registry-gateway.git
cd mcp-registry-gateway
cp .env.example .env
# Edit .env with your configuration

# Quick start with pre-built images
./build_and_run.sh --prebuilt

# Or build locally
./build_and_run.sh
```

## Breaking Changes from v1.x

This release establishes a new baseline. If upgrading from a previous fork:
- Review environment variable changes in `.env.example`
- Check authentication provider configuration
- Update any custom integrations to use the new API endpoints
